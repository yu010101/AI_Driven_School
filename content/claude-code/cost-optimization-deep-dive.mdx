# Claude Codeのコスト管理・最適化ガイド

「Claude Codeって月いくらかかるの？」は最もよく聞かれる質問です。この記事では料金体系の解説と、コストを最小限に抑えるテクニックを紹介します。

## 料金体系

Claude Codeには2つの利用方法があります:

### 1. Claude Pro/Max サブスクリプション

| プラン | 月額 | 使用量 |
|-------|------|-------|
| Claude Pro | $20/月 | 一定量まで（ヘビー利用は制限あり） |
| Claude Max 5x | $100/月 | Pro の5倍 |
| Claude Max 20x | $200/月 | Pro の20倍 |

**メリット**: 定額なので予算管理しやすい。個人や小チーム向け。

### 2. API 従量課金

| モデル | 入力 (1Mトークン) | 出力 (1Mトークン) |
|-------|-------------------|-------------------|
| Claude Sonnet | $3 | $15 |
| Claude Opus | $15 | $75 |

**1トークン ≈ 日本語1文字**（英語は約4文字で1トークン）

### 利用パターン別の月額シミュレーション

| パターン | 内容 | Pro | API (Sonnet) |
|---------|------|-----|-------------|
| ライト | 日に2-3回、簡単な質問 | $20 | $5-15 |
| ミドル | 日に10回、コーディング | $20 | $30-80 |
| ヘビー | 終日利用、大規模開発 | 制限に当たる | $100-300 |
| チーム5人 | ミドル×5 | $100 | $150-400 |

## トークン最適化テクニック10選

### 1. CLAUDE.mdは簡潔に
CLAUDE.mdは毎回の会話で読み込まれます。500行のCLAUDE.mdは毎回数千トークンを消費。100行以内に収めましょう。

### 2. `/clear` を定期的に使う
会話が長くなるとトークン消費が加速します。タスク完了ごとに `/clear` でリセット。

### 3. ファイル全体ではなく一部を読ませる
```
# 悪い例（ファイル全体を読む）
claude "app.tsxを修正して"

# 良い例（該当部分だけ指定）
claude "app.tsx の50-80行目のhandleSubmit関数を修正して"
```

### 4. 出力フォーマットを指定する
「詳しく説明して」ではなく「箇条書き5項目で」。出力トークンは入力の5-15倍高いです。

### 5. Sonnetを使う（Opusは5倍高い）
コーディング作業の90%はSonnetで十分。Opusは複雑な設計判断やレビューに限定。

### 6. キャッシュを活用する
同じファイルを何度も読ませない。CLAUDE.mdと大きなファイルはPrompt Cachingの恩恵を受けます。

### 7. 画像の送信を避ける
スクリーンショットは大量のトークンを消費。テキストで説明できるならテキストで。

### 8. バッチ処理は1回にまとめる
10ファイルを1つずつ修正するより、「以下の10ファイルをまとめて修正して」の方が効率的。

### 9. 生成されたコードをコピペで再利用
似たパターンのコードは、以前の生成結果をベースに修正を依頼。ゼロから生成させない。

### 10. 利用状況を定期的に確認する
Anthropic Consoleのダッシュボードで実際の使用量を確認。想定外の消費に早めに気づく。

## 競合との料金比較

| ツール | 月額 | 特徴 |
|-------|------|------|
| Claude Code (Pro) | $20 | ターミナル統合、プロジェクト全体理解 |
| GitHub Copilot | $10 | エディタ内補完、軽量 |
| Cursor Pro | $20 | VSCode fork、ファイル編集UI |
| Windsurf | $15 | Cursor類似、コスパ重視 |

Claude Codeの強みは「プロジェクト全体のコンテキスト理解」と「ターミナルでのフルスタック操作」です。単純なコード補完だけならGitHub Copilotの方が安くて適切です。

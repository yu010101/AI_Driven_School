## Claude Code の料金体系

Claude Codeを使う方法は3つあります。

| プラン | 料金 | 特徴 |
|--------|------|------|
| API従量課金 | 使った分だけ | 柔軟だがコスト管理必要 |
| Claude Pro | $20/月 | 個人向け、制限あり |
| Claude Max | $100/月 | ヘビーユーザー向け |

## 1. API従量課金

### 料金表

| モデル | 入力 | 出力 |
|--------|------|------|
| Claude Sonnet | $3/100万トークン | $15/100万トークン |
| Claude Opus | $15/100万トークン | $75/100万トークン |

※ 1トークン ≈ 日本語0.5文字、英語4文字

### コスト計算例

**ケース1: 簡単なタスク（15分の作業）**
```
入力: 約5,000トークン（コード読み込み + 指示）
出力: 約2,000トークン（回答 + 生成コード）

コスト（Sonnet）:
入力: 5,000 × $3 / 1,000,000 = $0.015
出力: 2,000 × $15 / 1,000,000 = $0.03
合計: 約$0.05（約7円）
```

**ケース2: 中規模タスク（1時間の作業）**
```
入力: 約50,000トークン
出力: 約20,000トークン

コスト（Sonnet）:
入力: $0.15
出力: $0.30
合計: 約$0.45（約65円）
```

**ケース3: 大規模タスク（1日の開発）**
```
入力: 約500,000トークン
出力: 約200,000トークン

コスト（Sonnet）:
入力: $1.50
出力: $3.00
合計: 約$4.50（約650円）
```

### 月額シミュレーション

| 使用頻度 | 月額目安（Sonnet） |
|----------|-------------------|
| ライト（1日30分） | $15-30 |
| ミドル（1日1-2時間） | $30-60 |
| ヘビー（1日3時間以上） | $60-150 |

### メリット・デメリット

**メリット:**
- 使った分だけ支払い
- 使わない月は0円
- Opusも使える

**デメリット:**
- コスト予測が難しい
- うっかり高額になる可能性
- APIキーの管理が必要

## 2. Claude Pro（$20/月）

### 含まれるもの

- Claude.aiでの対話（制限あり）
- Claude Codeの利用（制限あり）
- 優先アクセス

### 制限内容

- 1日のメッセージ数に上限
- 長い会話は途中でリセットが必要
- ピーク時は待ち時間あり

### 向いている人

- 個人開発者
- 学習目的
- 軽めの利用

### メリット・デメリット

**メリット:**
- 固定料金で安心
- APIキー不要
- Claude.aiも使える

**デメリット:**
- 制限を超えると使えない
- ヘビーユースには不向き

## 3. Claude Max（$100/月）

### 含まれるもの

- Claude Pro の全機能
- 5倍の利用量
- 優先アクセス（高速）

### 向いている人

- フルタイム開発者
- 1日数時間以上使う
- 待ち時間を避けたい

### メリット・デメリット

**メリット:**
- 実質使い放題に近い
- 高速レスポンス
- コスト予測可能

**デメリット:**
- 月$100は高め
- 軽い利用には割高

## どれを選ぶべき？

### フローチャート

```
毎日使う？
├── いいえ → API従量課金
└── はい
    ├── 1日1時間未満 → Claude Pro
    └── 1日1時間以上
        ├── コスト重視 → API従量課金
        └── 快適さ重視 → Claude Max
```

### 具体的なおすすめ

**学習・趣味利用:**
→ **Claude Pro**（$20/月）

理由:
- 固定料金で安心
- 学習には十分な量
- APIキー管理不要

**副業・フリーランス:**
→ **API従量課金** または **Claude Pro**

理由:
- 案件によって使用量が変動
- 使わない時は費用0
- 経費として計上しやすい

**フルタイム開発:**
→ **Claude Max**（$100/月）

理由:
- 制限を気にせず使える
- 待ち時間が少ない
- 時給換算で十分ペイする

**チーム開発:**
→ **各自API** または **チーム向けプラン**

理由:
- 個人のAPIキーで利用
- コストを各自管理
- または企業向けプランを検討

## コスト最適化のコツ

### コツ1: コンテキストを管理

```bash
# 長い会話はコストがかさむ
# 定期的にリセット
/clear
```

### コツ2: Sonnetを使う

OpusはSonnetの5倍のコスト。多くのタスクはSonnetで十分。

```bash
# モデルを指定（デフォルトはSonnet）
claude --model sonnet
```

### コツ3: 必要な時だけ使う

```
❌ 「ついでにこれも聞いておこう」
✅ 必要なタスクだけ依頼
```

### コツ4: プロジェクト設定を活用

CLAUDE.mdを設定することで、毎回の説明が不要に：

```markdown
# CLAUDE.md
TypeScript + Next.js プロジェクト
...
```

### コツ5: Subagentを適切に使う

Subagentは別のコンテキスト＝別のAPI呼び出し。

```
✅ 大規模調査 → Subagent使用（メインコンテキスト節約）
❌ 小さいタスク → Subagent不要
```

## よくある質問

### Q: 無料で試せる？

**A: Claude Pro の無料トライアル**

新規ユーザーは一定期間無料で試せます。

### Q: 請求のタイミングは？

**A: プランによる**

- API: 月末締め翌月払い
- Pro/Max: 月額サブスクリプション

### Q: 途中でプラン変更できる？

**A: はい**

- API ↔ Pro/Max は自由に切り替え可能
- Pro → Max もいつでも可能

### Q: チームで使う場合は？

**A: 各自でAPIキー、または企業向けプラン**

- 小規模: 各自API従量課金
- 大規模: Anthropic に相談

## まとめ

| プラン | 料金 | おすすめユーザー |
|--------|------|-----------------|
| API従量課金 | 使った分 | 不定期利用、コスト管理したい人 |
| Claude Pro | $20/月 | 個人開発者、学習目的 |
| Claude Max | $100/月 | フルタイム開発者 |

**選び方:**
1. まず Claude Pro で試す
2. 制限が気になったら Max を検討
3. 使用量にムラがあるなら API へ

**コスト最適化:**
- コンテキストを定期的にリセット
- Sonnetを使う（Opusは5倍高い）
- CLAUDE.md で毎回の説明を省略

## 自社ルールをスキル化する

公式スキルだけじゃない。**自社の業務ルールもスキルにできます。** CLAUDE.mdの強化版で、再利用可能な手順書をAIに覚えさせます。

:::before-after{before="毎回ルールを説明" after="/estimate で一発"}

### SKILL.mdの完全テンプレート

以下がSKILL.mdの基本構造です。このテンプレートをベースに自社スキルを作りましょう。

```markdown
# estimate（見積もり作成スキル）

## 概要
クライアント向け見積もりを会社の標準フォーマットで自動生成する。

## トリガー
`/estimate` コマンドで起動

## ルール
- 金額はすべて税込表記（消費税10%）
- 納期は実工数 + 3営業日のバッファを加算
- 端数は切り捨て（1円未満）
- 単価は職種別レートを適用（エンジニア: ¥80,000/日、デザイナー: ¥70,000/日）
- 有効期限は発行日から30日間

## 出力フォーマット
- Markdown形式の見積書
- 件名: [クライアント名]_見積書_[日付]
- 項目ごとに単価 × 数量 = 小計を表示
- 最下部に合計金額（税込）を表示

## 例
入力: "ABC社のLP制作、デザイン3日+実装5日"
出力: 合計 ¥924,000（税込）
```

この定義があれば、AIは毎回同じ品質の見積もりを出力します。見積もり以外にも、**請求書・契約書チェック・議事録**など「毎回同じ説明をしている作業」はすべてスキル化の候補です。

## SKILL.mdで業務を型化する

```
claude "うちの見積もり作成ルールをSKILL.mdに書いて。税込表記、納期は+3日バッファ、端数切り捨て"
```

SKILL.mdの構造はシンプル。**名前・説明・トリガー・手順**の4つだけ。一度作れば、新人でもベテランと同じ品質の見積もりが出せます。

### 作ったスキルをテストする

スキルを作成したら、必ず動作確認しましょう。

1. **基本テスト** — `/estimate` を実行して出力を確認
2. **エッジケース** — 端数が出る金額で切り捨てが正しく動くか
3. **フォーマット確認** — 税込表記・納期バッファが反映されているか

期待通りの出力が出なければ、SKILL.mdのルールを修正して再テスト。このサイクルを2〜3回回せば実用レベルになります。

### Gitでバージョン管理する

SKILL.mdはただのテキストファイルなので、Gitで管理できます。

```
git add .claude/skills/estimate/SKILL.md
git commit -m "feat: 見積もりスキルv1.0 — 税込表記・バッファ3日ルール追加"
```

変更履歴が残るので「先月のルールに戻したい」というときもすぐに対応できます。ルール変更のたびにコミットする習慣をつけましょう。

### チームで共有する

作ったスキルはチーム全員で使えます。

```
git push origin main
```

チームメンバーが `git pull` するだけで、同じスキルが全員のClaude Codeで使えるようになります。「ベテランの暗黙知」がSKILL.mdという形でチーム全体の資産になるのです。

右のターミナルにコピペして Enter を押してください。

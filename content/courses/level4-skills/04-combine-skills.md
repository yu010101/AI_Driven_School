## スキル同士を連携させる

:::flow{steps="💰 /cfo 売上データ → 📊 /cmo アクセスデータ → 📋 統合レポート"}

1つのスキルだけでも強力ですが、**組み合わせると部署横断の仕事を一瞬で終わらせます。** 経理とマーケの仕事をAIが同時にこなします。

### マルチスキル呼び出しの構文

スキルを組み合わせるには、1つのプロンプトに複数のスキル名を含めます。

```
claude "/cfo 月次レポートを作って、/cmo のGA4データも含めて"
```

Claude Codeは指示文中のスキル名（`/cfo`、`/cmo`）を自動認識し、それぞれのSKILL.mdを読み込んで処理します。明示的に「2つのスキルを使って」と言う必要はありません。

### スキルはどのように合成されるか

複数スキルを呼び出すと、以下の順序で処理されます。

1. **各スキルのSKILL.mdを読み込み** — 定義されたルールを理解
2. **データ取得フェーズ** — 各スキルが担当するデータを取得（CFO→売上、CMO→アクセス）
3. **統合フェーズ** — 取得したデータを1つのレポートに統合
4. **出力フェーズ** — 統合レポートを生成して保存

これが「CFOデータ → CMO分析」の流れです。CFOスキルが出した数字を、CMOスキルが読み取ってマーケティング視点で分析するイメージです。

### 3スキル連携の実践例

実務で使える3スキルワークフローの例です。

```
claude "/cfo 今月の売上データを取得して、/cmo のアクセス解析と照合して、/ceo 向けの経営サマリーを作って"
```

| フェーズ | スキル | 処理内容 |
|---------|--------|---------|
| ① データ収集 | `/cfo` | freeeから売上・経費・利益を取得 |
| ② 分析 | `/cmo` | GA4からPV・CVR・流入元を取得し売上と相関分析 |
| ③ レポート | `/ceo` | 経営者向けに要点をまとめたダッシュボードを生成 |

1コマンドで経理 → マーケ → 経営の3部署分の仕事が完了します。

## 部署を跨いだ仕事をAIに任せる

```
claude "/cfo 月次レポートを作って、/cmo のGA4データも含めて"
```

CFOスキルがfreeeから売上を取得し、CMOスキルがGA4からアクセスデータを取得。**2つの部署の仕事を1コマンドで統合**できます。人間がやると半日かかる月次レポートが30秒で完成します。

### 注意：コンテキストウィンドウの制約

スキルを組み合わせるほど、Claude Codeが処理する情報量は増えます。

- **2スキル** — 問題なし。ほとんどのケースで快適に動作
- **3スキル** — 通常は問題なし。データ量が多い場合は要注意
- **4スキル以上** — コンテキストウィンドウの上限に近づく可能性あり

大量データを扱うスキルを3つ以上組み合わせると、処理が不安定になることがあります。その場合は**2回に分けて実行**するのが確実です。

### 効率的なマルチスキル活用のコツ

1. **具体的に指示する** — 「レポート作って」より「4月の月次レポートをPL形式で」
2. **出力形式を指定する** — 「Markdown形式で」「表形式で」と明示する
3. **段階的に複雑にする** — まず2スキルで成功してから3スキルに挑戦
4. **結果を確認してから次へ** — 中間出力を確認する習慣をつける

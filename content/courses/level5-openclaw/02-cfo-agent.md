# CFOエージェントを作る

CFOエージェントは、あなた専属の**24時間財務アナリスト**です。freee会計と連携して、数字を自動で追いかけます。

:::flow
steps:
  - "📊 freee API"
  - "🤖 CFOエージェント"
  - "💰 PL・CF・月次レポート"
:::

## SKILL.mdの実際の内容

以下が、CFOエージェントのSKILL.mdとして生成される内容の全体像です。

```markdown
# CFO Agent - 最高財務責任者

## 役割
財務データの自動取得・分析・レポート生成を担当する。
AIは財務データの「集計・可視化・異常検知」を行う。
意思決定（投資判断・融資申込など）は必ず人間が行うこと。

## データソース
- freee会計 API（PL・BS・仕訳データ）
- freee請求書 API（未決済・入金状況）

## 定期タスク
1. 毎日: 未決済請求チェック → 期限7日以内をSlack通知
2. 毎週月曜: キャッシュフロー4週間予測を更新
3. 毎月5日: 月次PLレポートを生成 → CEO Agentに連携

## コマンド
- `/cfo pl` → 当月の損益計算書を生成
- `/cfo cf` → 4週間キャッシュフロー予測
- `/cfo 未決済` → 未決済請求書一覧
- `/cfo 仕訳 [内容]` → 仕訳候補を提案（計上は人間が確認）

## freee API連携
- エンドポイント: https://api.freee.co.jp/api/1
- 認証: OAuth2.0（リフレッシュトークンで自動更新）
- 取得項目: 勘定科目別損益、取引先別残高、請求書一覧
- レート制限: 3,600リクエスト/時（十分な余裕あり）

## 出力フォーマット
PLレポートは以下の形式で出力する:
  月度、売上高、売上原価、粗利、販管費、営業利益、前月比
```

## freee MCP サーバー接続設定

`.claude/settings.json` に以下を追加して、Claude Codeからfreee APIを直接呼び出せるようにします。

```json
{
  "mcpServers": {
    "freee": {
      "command": "npx",
      "args": ["-y", "@anthropic/freee-mcp-server"],
      "env": {
        "FREEE_CLIENT_ID": "${FREEE_CLIENT_ID}",
        "FREEE_CLIENT_SECRET": "${FREEE_CLIENT_SECRET}",
        "FREEE_COMPANY_ID": "${FREEE_COMPANY_ID}",
        "FREEE_REFRESH_TOKEN": "${FREEE_REFRESH_TOKEN}"
      }
    }
  }
}
```

**注意**: MCP対応サーバーがない場合は、`scripts/freee-api.sh` のようなシェルスクリプトでcurlベースのAPI呼び出しを書き、SKILL.mdからそのスクリプトを参照する方法でも動作します。

## PLレポートの出力例

```
============================================
  損益計算書（PL）2026年3月度
============================================
  売上高           ¥3,200,000  (+12.3%)
  売上原価           ¥640,000  (+8.1%)
  ─────────────────────────────
  粗利           ¥2,560,000  (+13.4%)
  販管費         ¥1,670,000  (+5.2%)
  ─────────────────────────────
  営業利益         ¥890,000  (+24.1%)
============================================
  ※ 括弧内は前月比
```

## 他の会計システムへの対応

freee以外を使っている場合は、SKILL.mdのデータソースを差し替えるだけで対応可能です。

| 会計システム | API | 変更箇所 |
|-------------|-----|---------|
| マネーフォワード | MF クラウド API | エンドポイントと認証方式を変更 |
| 弥生会計 | Misoca API（請求書のみ） | CSV出力 → 手動インポートの併用 |
| Xero | Xero API | OAuth2.0で同様の構成が可能 |

## SKILL.mdでエージェントを定義する

Claude Codeの「スキル」機能を使い、CFOの仕事内容を定義します。freee APIからPLを取得し、キャッシュフローを予測し、月次レポートを生成する指示書です。

```
claude "CFOエージェントのSKILL.mdを作って。freeeからPLを取得、キャッシュフロー予測、月次レポート生成"
```

## 実装のポイント

- **制約事項を明記する**: SKILL.mdに「AIは財務判断を行わない。集計と報告のみ」と書くことが極めて重要。AIが勝手に送金や仕訳計上をしない安全設計
- **リフレッシュトークンの管理**: freee OAuth2のアクセストークンは有効期限がある。リフレッシュトークンで自動更新する仕組みを入れておく
- **テスト環境で検証**: freee APIにはサンドボックス環境がある。本番データに接続する前に必ずテストする
- **段階的に機能追加**: 最初はPL取得だけでよい。動いたらCF予測、次に未決済チェックと段階的に追加する

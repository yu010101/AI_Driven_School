## 🏆 認定模擬テスト ④ — Agent Skills

**問題：Claude Codeで利用可能なAgent Skillsを検索するコマンドは？**

Skillsは SKILL.md で定義する拡張モジュールです。

:::flow{steps="🔍 スキル検索 → 📦 インストール → ⚡ 即座に利用可能"}

> SKILL.md形式はAntigravity等、他のAI IDEでも互換性があります。

### 追加練習問題

**Q1：公式スキルをインストールするコマンドは？（例：SEOスキル）**

A：`claude /install-skill seo` — `/install-skill` の後にスキル名を指定します。インストール後は `.claude/skills/` フォルダにSKILL.mdが配置され、`/seo` というトリガーで呼び出せるようになります。アンインストールは該当フォルダを削除するだけです。

**Q2：SKILL.mdに最低限必要な4つの要素は何ですか？**

A：(1) 名前 — スキルの識別名、(2) 説明 — 何をするスキルか、(3) トリガー — 呼び出しコマンド（例：`/estimate`）、(4) 手順 — AIが実行するルールと出力フォーマット。この4要素が揃っていれば最小限のスキルとして機能します。より精度を上げるには、入力例・出力例・エッジケースの定義を追加します。

**Q3：SKILL.mdが他のAI IDE（Cursor、Antigravity等）でも動く理由は？**

A：SKILL.mdはただのMarkdownテキストファイルだからです。特定のIDEに依存するバイナリ形式や独自フォーマットではないため、Markdownを読めるAI IDEならどれでも解釈できます。ビジネスロジック（ルールや手順）はIDE非依存なので、同じSKILL.mdがどの環境でも同じ結果を返します。

### Agent Skills認定 学習のコツ

- **スキルのライフサイクルを理解する** — 検索 → インストール → 利用 → アンインストール
- **SKILL.mdの構造を書けるようにする** — 4要素を空で書ける程度に暗記
- **ポータビリティの概念** — 「なぜどのIDEでも動くのか」を説明できるように

### よくある間違い

- `/install-skill` と `/find-skills` を混同する → find は検索、install はインストール
- SKILL.mdをCLAUDE.mdの中に書く → 別ファイルとして `.claude/skills/` フォルダに配置する
- スキルの組み合わせ時にコンテキストウィンドウの限界を忘れる → 3スキル以上の同時使用は慎重に

## 練習問題

**Q1**: SKILL.mdのフォーマットが他のAI IDE（Cursor、Antigravityなど）でも動作する理由として正しいのはどれか？
A) 各IDEが専用のコンバーターを内蔵している  B) SKILL.mdはただのMarkdownテキストなので、Markdownを読めるAIならどれでも解釈できる  C) Anthropicが全IDEにライセンス供与している  D) バイナリ互換性が保証されている
→ 正解: B。SKILL.mdは特定のIDEに依存しないプレーンなMarkdownファイルです。中身はビジネスロジック（ルールと手順）の自然言語記述なので、Markdownを読めるAI環境ならどこでも同じように動作します。

**Q2**: SKILL.mdに最低限必要な4つの要素の正しい組み合わせはどれか？
A) 名前・バージョン・ライセンス・依存関係  B) 名前・説明・トリガー・手順  C) タイトル・日付・著者・本文  D) 入力・処理・出力・エラーハンドリング
→ 正解: B。名前（識別名）、説明（何をするか）、トリガー（`/estimate` のような呼び出しコマンド）、手順（AIが実行するルールと出力フォーマット）の4つが揃えば最小限のスキルとして機能します。

**Q3**: スキルをインストールした後の配置場所として正しいのはどれか？
A) `node_modules/` フォルダ  B) `.claude/skills/` フォルダ  C) `/usr/local/bin/`  D) `package.json` の中
→ 正解: B。`/install-skill` でインストールしたスキルは `.claude/skills/` フォルダにSKILL.mdとして配置されます。アンインストールは該当フォルダを削除するだけです。

**Q5**: 複数のスキルを同時に使用する際の注意点として正しいのはどれか？

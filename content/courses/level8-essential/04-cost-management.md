## コスト管理 — 料金を把握して使いすぎを防ぐ

経営者が最初に聞くこと＝「**いくらかかるの？**」。料金体系を理解して賢く使おう。

:::before-after{before="気づいたら高額請求" after="使用量を見える化"}

Claude Pro は $20/月の固定料金。API は従量課金で使い方により大きく変わる。

### 料金体系の全体像

Claude Codeの利用には主に2つの課金形態があります：

| プラン | 料金 | 特徴 |
|---|---|---|
| **Claude Pro（サブスク）** | $20/月 | 一定量まで使い放題。上限到達で速度制限 |
| **Claude Max（サブスク）** | $100/月 or $200/月 | Pro の5倍〜20倍の使用量。ヘビーユーザー向け |
| **API従量課金** | 使った分だけ | 完全従量制。大量利用時はこちらが安い場合も |

### APIトークン単価（2025年時点）

| モデル | 入力 (per 1M tokens) | 出力 (per 1M tokens) | キャッシュ入力 |
|---|---|---|---|
| **Claude Opus 4** | $15.00 | $75.00 | $1.88 |
| **Claude Sonnet 4** | $3.00 | $15.00 | $0.30 |

**重要**: 出力トークンは入力の5倍の単価。Claudeに長い回答を求めるほどコストが上がります。

### 月額コストシミュレーション

**ライトユーザー（週2〜3回利用、簡単な質問）**
```
Sonnet 4 使用
入力: 500K tokens × $3.00/1M = $1.50
出力: 100K tokens × $15.00/1M = $1.50
月額合計: 約 $3〜5（約450〜750円）
→ Pro $20/月より安い。APIの方がお得
```

**ミドルユーザー（毎日利用、コーディング補助）**
```
Sonnet 4 使用
入力: 5M tokens × $3.00/1M = $15.00
出力: 1M tokens × $15.00/1M = $15.00
月額合計: 約 $30（約4,500円）
→ Pro $20/月の方がお得
```

**ヘビーユーザー（終日利用、大規模開発）**
```
Opus 4 使用
入力: 20M tokens × $15.00/1M = $300.00
出力: 4M tokens × $75.00/1M = $300.00
月額合計: 約 $600（約90,000円）
→ Max $200/月が圧倒的にお得
```

### 競合サービスとのコスト比較

| サービス | 月額 | 特徴 |
|---|---|---|
| **Claude Code（Pro）** | $20/月 | ターミナルベース、高い自律性 |
| **Claude Code（Max）** | $100〜200/月 | 大量利用向け |
| **GitHub Copilot** | $10/月 | エディタ統合、コード補完特化 |
| **Cursor Pro** | $20/月 | VS Code fork、AI統合エディタ |
| **Cursor Ultra** | $200/月 | 無制限に近い利用 |
| **Windsurf** | $15/月 | VS Code fork、フロー重視 |

**判断基準**: Copilot $10/月はコード補完のみ。Claude Codeはファイル操作・git操作・テスト実行まで自律的に行える。単純な月額比較ではなく「何を自動化できるか」で判断すること。

### コスト最適化チェックリスト（10項目）

1. **モデルを使い分ける** — 簡単なタスクはSonnet、複雑な設計はOpus
2. **`/compact` を定期実行** — Context圧縮で入力トークンを80%削減
3. **CLAUDE.mdを簡潔に** — 50行以内。不要な説明は削除
4. **ファイルは部分読み込み** — 全体ではなく必要な関数だけ読む
5. **1セッション1タスク** — タスク完了ごとに新セッション開始
6. **出力を制御する** — 「簡潔に回答して」と指示すれば出力トークン削減
7. **キャッシュを活用** — 同じファイルの再読み込みはキャッシュ単価（1/10）で済む
8. **不要なツール実行を減らす** — 明確な指示で無駄な探索を防ぐ
9. **使用量ダッシュボードを週1確認** — 異常な増加を早期発見
10. **チームでProプランを共有しない** — 1アカウント1人。共有すると上限が即到達

### 使用量ダッシュボードの読み方

```
💰 API使用量レポート（今月）

期間: 2025/04/01 - 04/09

┌────────────┬──────────┬─────────┐
│ 項目        │ 使用量    │ コスト   │
├────────────┼──────────┼─────────┤
│ 入力トークン │ 2.1M     │ $6.30   │ ← あなたの指示 + ファイル内容
│ 出力トークン │ 420K     │ $6.30   │ ← Claudeの回答
│ キャッシュ   │ 1.8M     │ $0.54   │ ← 再利用された入力（安い）
├────────────┼──────────┼─────────┤
│ 合計        │          │ $13.14  │
└────────────┴──────────┴─────────┘
```

**注目すべき指標：**
- **入出力比率**: 出力が入力の50%以上なら、Claudeに冗長な回答をさせている可能性
- **キャッシュ率**: キャッシュが高いほどコスト効率が良い（同じ情報の再利用）
- **日別推移**: 急増した日があればその日の作業を振り返る

### 実務での使い方

**個人開発者**: Pro $20/月から始める。上限に頻繁に当たるようならMax $100/月に移行。APIは予算管理が難しいので上級者向け。

**スタートアップ（3〜5人）**: 各メンバーにPro契約。共通のCLAUDE.mdで利用ルールを統一。月次でチーム全体のコストレビュー。

**企業（10人以上）**: API従量課金 + 月額上限設定。管理者がダッシュボードで全員の利用量をモニタリング。部門別の予算配分。

### よくある失敗

**1. Opusを常用して請求額が跳ね上がる**
Opusの出力単価はSonnetの5倍。「とりあえずOpus」ではなく、タスクの複雑さに応じて使い分ける。

**2. 長い会話を放置して入力トークンが膨張**
会話が長くなるほど、毎ターンの入力トークンが増える。`/compact` せずに100ターン続けると、後半は1ターンで10万トークン以上消費する場合も。

**3. エラーループで無駄にトークンを消費**
Claudeが同じエラーを繰り返し修正しようとする場合、手動で方向修正した方がトークン効率が良い。

**4. 無料枠があると思い込む**
Claude Code にはClaude.aiのような無料枠は存在しない。利用開始時点からProプランまたはAPI課金が必要。

## コストを確認しよう

コスト最適化の3原則：

- **モデルを使い分ける** — Sonnet（日常）vs Opus（複雑なタスク）
- **Context を圧縮する** — `/compact` で入力トークンを80%削減
- **出力を制御する** — 簡潔な回答を求めて出力トークンを節約

月額の目安: ライト $3〜5、ミドル $20（Pro推奨）、ヘビー $100〜200（Max推奨）

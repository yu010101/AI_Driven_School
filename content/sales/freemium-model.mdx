# フリーミアムモデルの設計

無料ユーザーを増やしても、有料に転換しなければ意味がない。

## フリーミアムの成功条件

### 1. 大きな市場

- 無料ユーザーの1-5%しか課金しない
- 1万人の無料ユーザー → 100〜500人の有料ユーザー

### 2. 低い限界コスト

- ユーザーが増えてもコストが大きく増えない
- サーバー代が膨らむなら要注意

### 3. 明確な価値の差

- 無料で「あ、いいな」
- 有料で「これがないと困る」

## 機能の分け方パターン

### パターン1: 使用量制限

```
無料: 月100回まで
有料: 無制限
```

例: API、ストレージ、エクスポート数

### パターン2: 機能制限

```
無料: 基本機能のみ
有料: 高度な機能
```

例: レポート、連携、カスタマイズ

### パターン3: チーム制限

```
無料: 1人まで
有料: チームで利用可能
```

例: コラボレーション、権限管理

### パターン4: サポート制限

```
無料: コミュニティサポート
有料: 優先サポート
```

## 悪い境界線の例

### 無料が貧弱すぎる

- 価値を体験できない
- 口コミが広がらない
- 「使えない」で離脱

### 無料が充実しすぎ

- 課金する理由がない
- 「これで十分」で終わる
- 収益化できない

## 転換率を上げるコツ

### 1. 価値を先に体験させる

- 無料トライアルで全機能開放
- 「これがなくなるのは困る」状態を作る

### 2. 制限に近づいたら通知

```
「今月の無料枠の80%を使用しました」
→ アップグレードへの導線
```

### 3. 成功体験の直後に提案

- タスク完了時
- 目標達成時
- 「もっとやりたい」時

### 4. 年払い割引

```
月払い: 1,980円/月
年払い: 15,800円/年（2ヶ月無料）
```

## 成功事例から学ぶ

### Notion

- 無料: 個人利用は無制限
- 有料: チーム機能、高度なブロック
- **ポイント**: 個人で広がり、チームで課金

### Canva

- 無料: 基本テンプレート
- 有料: プロ素材、背景削除
- **ポイント**: 無料で十分使えるが、プロ機能が魅力的

### Slack

- 無料: 直近90日のメッセージ
- 有料: 全履歴、連携無制限
- **ポイント**: 使い続けると有料が必要になる

## 指標の目安

| 指標 | 目安 |
|-----|-----|
| 無料→有料転換率 | 2-5% |
| トライアル→有料 | 15-25% |
| 月次解約率 | 3-7% |

## 実装チェックリスト

- [ ] 無料で価値を体験できる
- [ ] 有料にする明確な理由がある
- [ ] 制限はストレスではなく「もっと欲しい」
- [ ] アップグレード導線が自然
- [ ] 指標を計測する仕組みがある

# 紹介プログラムの設計

最も信頼される広告は「友人からの推薦」。

## 紹介プログラムのメリット

### 高い転換率

- 広告経由: 1-2%
- 紹介経由: 10-30%

### 低いCAC

- 広告: 顧客獲得コストが高騰
- 紹介: 報酬コストのみ

### 高いLTV

- 紹介経由の顧客は解約率が低い
- 友人と一緒に使うので継続する

## 報酬設計

### 報酬の種類

| タイプ | 例 | 特徴 |
|-------|-----|------|
| 現金 | 1,000円キャッシュバック | シンプルで魅力的 |
| クレジット | 1ヶ月無料 | サービス継続につながる |
| 機能解放 | プレミアム機能開放 | コストがかからない |
| 商品 | オリジナルグッズ | ファン向け |

### 誰に報酬を渡すか

```
紹介者のみ: 紹介モチベーション高
被紹介者のみ: 登録ハードル低
両方: バランス型（推奨）
```

### 報酬の目安

```
LTV × 20-30% = 紹介報酬の上限
```

例：
- LTV 50,000円 → 報酬上限 10,000〜15,000円
- 月額1,000円のサービス → 1-2ヶ月分が目安

## 仕組みの作り方

### シンプルな仕組み

```
1. 紹介コードを発行
2. 友人がコードを使って登録
3. 両者に報酬を付与
```

### 必要な機能

- 紹介コード/リンクの生成
- 紹介状況のトラッキング
- 報酬の自動付与
- 紹介状況の確認画面

### 実装方法（個人開発向け）

```javascript
// 簡易的な紹介コード生成
const referralCode = `REF_${userId}_${randomString(6)}`

// 登録時に紹介コードをチェック
if (referralCode) {
  // 紹介者を特定
  const referrer = await findUserByReferralCode(referralCode)
  // 両者に報酬を付与
  await addCredit(referrer.id, 1000)
  await addCredit(newUser.id, 1000)
}
```

## 紹介を促すタイミング

### ベストタイミング

1. **価値を実感した直後**
   - タスク完了時
   - 目標達成時
   - 「すごい！」と思った瞬間

2. **自然な会話が生まれる時**
   - 成果をシェアしたくなる時
   - 友人に聞かれた時

### 紹介の依頼方法

```
❌「友達を紹介してください」
→ 唐突すぎる

⭕「友達にも教えてあげませんか？
　 紹介すると、あなたも友達も1ヶ月無料になります」
→ メリットを明示
```

## 成功事例

### Dropbox

- 紹介で容量追加（両者に500MB）
- 成長率 3900%
- ユーザー獲得コストほぼゼロ

### Uber

- 紹介で無料乗車（両者に）
- 口コミで急成長
- 新市場開拓の主力

### Notion

- 紹介でクレジット付与
- 無料プランで広がり、チームで課金


### 3. 定期的にリマインド

```

```

## 計測すべき指標

| 指標 | 説明 |
|-----|-----|
| 紹介率 | ユーザー中、紹介した人の割合 |
| K-factor | 1人が何人連れてくるか |
| 紹介経由CVR | 紹介リンク経由の登録率 |
| 紹介経由LTV | 紹介経由顧客の生涯価値 |

### K-factor の計算

```
K = 招待数 × 転換率

例: 平均5人招待 × 20%転換 = 1.0
K > 1 ならウイルス的成長
```

# ポジショニングとタグライン

## サイトの一言定義（最重要）

**非エンジニアでも「作れて・動いて・売れる」をAIで実現する実践知のデータベース**

## 誰のためのサイトか

- 非エンジニア
- 個人開発したい人
- AIは触っているが「再現性」がない人
- 作ったけど売れない人

## 何が得られるか

1. **作り方**（バイブコーディング）
2. **実装の型**（機能・構成）
3. **売り方**（0円マーケ）

## トップページ構成（文章そのまま使える）

### Hero

```
非エンジニアでも、
AIで「作れて・動いて・売れる」。

バイブコーディング × 個人開発 × 0円マーケ
── 実践知を体系化した知識データベース
```

### サブコピー

```
・コードが書けなくてもOK
・設計で詰まらない
・作って終わりにしない

再現できる「型」だけを残しました。
```

### CTA

- 「無料で学ぶ：バイブコーディングの教科書」
- 「無料で学ぶ：0円マーケティングの教科書」

**※ 2ボタン併設しない → ページ文脈で出し分ける**

## 各DBのポジショニング

### バイブコーディングDB

**タグライン**: 「コードが書けなくても、AIで作れる」

**価値提案**: 
- 非エンジニアでも再現可能なプロンプト集
- 失敗例から学ぶ実践知
- チェックリストで漏れを防ぐ

### 実装DB

**タグライン**: 「作り方は分かった。実装で詰まらない」

**価値提案**:
- 機能単位の実装手順
- よくある詰まりの解決策
- 代替案（ノーコード含む）

### 0円マーケDB

**タグライン**: 「作ったら、0円で売れる」

**価値提案**:
- 無料でできる集客施策
- コピペで使えるテンプレ
- 成功・失敗パターン

# Cursor / Claude Code 用プロンプト集

## ①-1 前提注入プロンプト（最初に1回）

```
あなたはこのプロジェクト専属のシニアエンジニア兼SEO編集者です。

このサイトの目的は以下です。

- 非エンジニア向け
- AIを使って「作れて・動いて・売れる」状態を実現させる
- バイブコーディング / 個人開発 / 0円マーケの知識DB

制約条件：
- 初心者が再現できない説明は禁止
- 抽象論・精神論は禁止
- コピペで使える内容を必ず含める
- SEOを意識し、検索意図を最優先する
- 記事は体系的に内部リンクできる構造にする

以後の指示はすべてこの前提を守って実行してください。
```

**👉 Cursor / Claude Code 両方で最初に流す**

---

## ①-2 DB記事生成プロンプト（カテゴリ共通）

```
以下の条件でSEO記事を1本作成してください。

【カテゴリ】
{vibe-coding | build | marketing}

【狙う検索キーワード】
{キーワード}

【想定読者】
- 非エンジニア
- 初心者
- 個人開発・個人ビジネス志向

【必須構成】
1. 結論（30秒要約）
2. 初心者向け定義
3. なぜ重要か
4. 具体例（良い例・悪い例）
5. コピペで使える実用パート
6. よくある失敗
7. 次に読むべき記事（2〜3本想定で記述）

【トーン】
- 冷静
- 断定的
- 再現性重視

【禁止】
- 曖昧な表現
- 「人による」「ケースバイケース」
- 表面的な箇条書き

Markdown形式で出力してください。
```

**使い方**: `{カテゴリ}` と `{キーワード}` を実際の値に置き換えて使用

## ①-3 内部リンク最適化プロンプト

```
この記事をDBサイトの一部として最適化してください。

- 同カテゴリ内への内部リンク候補を3つ
- 別カテゴリへの橋渡しリンクを2つ
- 「次にやるべき行動」を明示

リンクは「アンカーテキスト案」として文章で提案してください。
```

**👉 これで"勝手にクラスター構造"が育つ**

## ①-4 Next.js（App Router）実装プロンプト

```
以下の構成でNext.js（App Router）プロジェクトを設計してください。

- /knowledge/vibe-coding
- /knowledge/build
- /knowledge/marketing
- slugベースの動的ルーティング
- MDX対応
- sitemap.ts / robots.ts 含む
- SEOを意識したmetadata生成

最小構成で、後からSupabaseに移行できる設計にしてください。
```

## ①-5 記事リライトプロンプト（成果が出た記事用）

```
以下の記事を、検索順位を上げるためにリライトしてください。

【現在の順位】
{順位}位

【狙うキーワード】
{キーワード}

【改善ポイント】
- 検索意図により応える
- より具体的な実例を追加
- 内部リンクを最適化
- 構造化データ（FAQ/HowTo）を追加

元の記事の価値は維持しつつ、SEOを強化してください。
```

---

## プロンプト使用フロー

### 1. 初期設定（1回だけ）

```
①-1の前提注入プロンプトを実行
```

### 2. 記事生成（毎回）

```
①-2の記事生成プロンプトに以下を入力：
- カテゴリ: vibe-coding
- キーワード: バイブコーディング とは

出力されたMarkdownを .mdx ファイルとして保存
```

### 3. 内部リンク追加（毎回）

```
生成した記事に①-3のプロンプトを適用
frontmatterに relatedArticles を追加
```

### 4. リライト（30日後）

```
Search Consoleで順位20位以内の記事を特定
①-5のリライトプロンプトで改善
```

---

## 注意事項

### ✅ やるべきこと

- 前提注入プロンプトは必ず最初に実行
- キーワードは具体的に指定
- 出力は必ず目視チェック（5分）

### ❌ やってはいけないこと

- プロンプトを省略しない
- 曖昧なキーワード指定
- AI出力をそのまま公開（最低限のチェックは必須）

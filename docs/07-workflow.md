# 記事生成〜公開までの運用フロー

## 全体フロー（人間はここだけやる）

```
① KW選定（人）
    ↓
② 記事生成（AI）
    ↓
③ 最低限の目視チェック（人：5分）
    ↓
④ Git push / Notion export（人）
    ↓
⑤ 公開・インデックス（自動）
```

---

## ②-1 KW管理（Notion or Sheets）

### 最低限のカラム

| slug | keyword | category | status | priority | created_at | published_at |
|------|---------|----------|--------|----------|------------|--------------|
| vibe-what | バイブコーディング とは | vibe-coding | todo | high | - | - |
| vibe-prompts | バイブコーディング プロンプト | vibe-coding | generated | high | 2024-01-01 | - |
| ... | ... | ... | ... | ... | ... | ... |

### status管理

- `todo` - 未着手
- `generated` - 生成済み（未チェック）
- `checked` - チェック済み（未公開）
- `published` - 公開済み

### priority管理

- `high` - 最優先（内部リンク起点になる記事）
- `medium` - 通常
- `low` - 後回しOK

---

## ②-2 記事生成（Cursor / Claude Code）

### ステップ

1. **KWを1つコピー**
   - Notion/Sheetsから `keyword` と `category` を取得

2. **プロンプトに貼る**
   ```
   【カテゴリ】
   vibe-coding
   
   【狙う検索キーワード】
   バイブコーディング とは
   ```

3. **出力を .mdx に保存**
   - ファイル名: `{slug}.mdx`
   - 保存先: `content/{category}/{slug}.mdx`

4. **frontmatterを微調整**
   - title, description を最適化
   - tags を追加
   - relatedArticles を設定

**⏱ 所要時間**: 1記事10〜15分

## ②-3 機械生成でもSEOで死なないチェックリスト（5分）

### 最低限これだけ見る

- [ ] タイトルが検索KWを含んでいる
- [ ] 冒頭で結論が出ている（30秒要約）
- [ ] コピペ可能な実用パートがある（プロンプト/テンプレ/コード）
- [ ] 内部リンク候補がある（relatedArticles）
- [ ] CTAは1つだけ（混在していない）

**※ リライトはしない（量と構造を優先）**

## ②-5 成果が出た後にやること（重要）

### 30日後に見る指標

1. **impressions が出ている記事**
   - 検索されている = 需要がある
   - 順位が低い場合はリライト候補

2. **平均掲載順位20位以内の記事**
   - 上位表示の可能性あり
   - リライトで1位を狙う

### リライトフロー

```
順位20位以内の記事を特定
    ↓
①-5のリライトプロンプトで改善
    ↓
記事を更新
    ↓
30日後に再評価
```

今回の構造は:
- 明確な検索意図
- 再現性
- 内部リンク網

→ 人間編集メディアと同等評価

### ❌ 反論: 「全部自動だと思想がブレる」

**✅ 止揚**: 
思想は **①の前提注入プロンプト** が担保

人間は「設計者」に徹するのが最適

## 運用のベストプラクティス

### ✅ やるべきこと

- **量より質** → でも最初は量も重要
- **チェックは5分だけ** → 完璧を目指さない
- **データを見る** → 30日後に必ず評価
- **リライトは順位20位以内だけ** → 無駄な作業を避ける

### ❌ やってはいけないこと

- 全記事を完璧にリライト（時間の無駄）
- データを見ずに感覚で判断
- チェックを省略（最低限は必須）
- 一度に全部作ろうとする（段階的に）

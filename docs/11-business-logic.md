# ビジネスロジック

## リードマグネット配置ルール

### 基本原則

**1ページに1つのCTAのみ**

### 配置ルール

#### バイブコーディングDB記事

- **CTA**: 「バイブコーディングの教科書」のみ
- **配置位置**: 記事末尾
- **文言**: 「【無料】バイブコーディングの教科書」
- **リンク先**: `/free/vibe-coding-book`

#### 実装DB記事

- **CTA**: 「バイブコーディングの教科書」のみ
- **配置位置**: 記事末尾
- **文言**: 「【無料】作れる人になる：バイブコーディングの教科書」
- **リンク先**: `/free/vibe-coding-book`

#### 0円マーケDB記事

- **CTA**: 「0円マーケティングの教科書」のみ
- **配置位置**: 記事末尾
- **文言**: 「【無料】0円マーケティングの教科書」
- **リンク先**: `/free/zero-marketing-book`

### 禁止事項

- ❌ 1ページに2つのCTAを同時に配置
- ❌ 同じ文言で両方のCTAを表示
- ❌ 記事途中にCTAを配置（末尾のみ）

## 内部リンクルール

### 基本原則

**すべての記事に「次の一手」を用意**

### リンク数

- **同カテゴリ内**: 最低3本
- **別カテゴリ**: 最低2本
- **合計**: 最低5本

### リンク配置

#### 記事本文内

- 自然な文脈でリンクを配置
- アンカーテキストは具体的に（「こちら」禁止）

#### 記事末尾

- 「次に読むべき記事」セクション
- 関連記事を3-5本表示

### リンクパターン

#### バイブコーディング → 実装

```
バイブコーディング記事末尾
→「次は ログイン機能 を作ろう」
→ `/knowledge/build/features/auth`
```

#### 実装 → マーケティング

```
実装記事末尾
→「作ったら Xで0円集客 しよう」
→ `/knowledge/marketing/tactics/x-thread`
```

#### マーケティング → バイブコーディング

```
マーケ記事末尾
→「次は 改善のために作り直す」
→ `/knowledge/vibe-coding/flows/improvement`
```

## 記事の必須要素

### 全記事共通

1. **結論（30秒要約）** - H1直下
2. **初心者向け定義** - 専門用語を避ける
3. **具体例** - 良い例・悪い例
4. **コピペ可能な実用パート** - プロンプト/テンプレ/コード
5. **よくある失敗** - 読者の不安を解消
6. **次にやるべきこと** - 内部リンクで導線
7. **CTA** - 1つだけ

### カテゴリ別追加要素

#### バイブコーディングDB

- プロンプトテンプレート（必須）
- 失敗例（必須）
- チェックリスト（推奨）

#### 実装DB

- 実装ステップ（必須）
- テスト観点（必須）
- 代替案（推奨）

#### マーケDB

- 手順（必須）
- 成功パターン（必須）
- 失敗パターン（必須）
- コピペ用テンプレ（必須）

## SEO要件

### メタデータ

- **title**: 検索KWを含む（60文字以内）
- **description**: 検索意図に応える（160文字以内）
- **OGP**: 全ページで設定

### 構造化データ

- **FAQ**: バイブコーディングDB記事に推奨
- **HowTo**: 実装DB記事に推奨
- **Article**: 全記事に設定

### 内部リンク

- **アンカーテキスト**: 具体的なキーワードを含む
- **関連性**: 意味のある関連記事のみリンク

## 記事ステータス管理

### ステータス

- `draft`: 下書き（非公開）
- `published`: 公開済み
- `archived`: アーカイブ（非公開）

### 公開ルール

- `published`のみ表示
- `draft`は開発環境のみ表示
- `archived`は非表示

## カテゴリ別ルール

### バイブコーディングDB

- **対象読者**: 非エンジニア、初心者
- **トーン**: 優しく、具体的に
- **禁止**: 専門用語の多用

### 実装DB

- **対象読者**: 個人開発者、非エンジニア
- **トーン**: 実践的、再現性重視
- **必須**: コード例、エラー対処

### マーケDB

- **対象読者**: 個人開発者、起業家
- **トーン**: 具体的、成果重視
- **必須**: 0円であることの明記

## バリデーションルール

### 記事作成時

- [ ] タイトルに検索KWを含む
- [ ] 結論が冒頭にある
- [ ] コピペ可能なパートがある
- [ ] 内部リンクが5本以上
- [ ] CTAが1つだけ
- [ ] メタデータが設定されている

### 公開前チェック

- [ ] ビジネスロジックに準拠
- [ ] SEO要件を満たす
- [ ] リンクが正しく動作
- [ ] 画像が最適化されている
- [ ] モバイル表示確認

## エラー処理

### 記事が見つからない場合

- **404ページ**: カスタム404を表示
- **メッセージ**: 「お探しの記事は見つかりませんでした」
- **代替案**: カテゴリ一覧へのリンク

### カテゴリが存在しない場合

- **リダイレクト**: `/knowledge` へ
- **ログ**: エラーを記録

### CTAリンクが無効な場合

- **フォールバック**: `/free` へリダイレクト
- **エラー通知**: 管理者に通知
